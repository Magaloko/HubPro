1. Executive Summary

Der Content Hub Pro ist eine hochspezialisierte Enterprise-SaaS-Plattform, die entwickelt wurde, um die fragmentierten Workflows moderner Content-Agenturen zu zentralisieren. Aktuell nutzen Agenturen oft eine Vielzahl isolierter Tools — Excel für die Planung, Word für die Redaktion, Later für Social Media, Neuronwriter für SEO. Der Content Hub Pro bündelt all diese Prozesse in einer kollaborativen Single-Page-Application (SPA).

Die Plattform zeichnet sich durch eine strikte Trennung von Social Media Management und SEO-Blog-Content aus, während sie gleichzeitig eine gemeinsame Datenbasis für Analytics und Kundenfreigaben nutzt. Durch die Integration von KI-gestützten Discovery-Tools und einem Track-Changes-Modus für Kunden wird die Effizienz in der Kommunikation zwischen Agentur und Klient um bis zu 40% gesteigert.

2. Product Vision

Die Vision des Content Hub Pro ist es, das Betriebssystem für Content-Marketing zu werden.

Vier strategische Säulen

SäuleBeschreibung
ZentralisierungEin einziger “Source of Truth” für alle Content-Assets. Keine Datensilos mehr zwischen Planung, Erstellung und Distribution.
Datengetriebene KreativitätIntegration von Keyword-Metriken und Social-Trends direkt in den Erstellungsprozess. Der SEO-Score wird live berechnet.
Transparente KollaborationEin Freigabeprozess, der Änderungen nachvollziehbar macht und Revisionen minimiert. Track-Changes-Modus für Kunden.
AutomatisierungOne-Click-Publishing von der Planung direkt in die Zielkanäle (WordPress, Instagram, LinkedIn).
Roadmap-Ziel: In Phase 3 wird die API-Integration für WordPress REST, Meta Graph API und LinkedIn API implementiert, um den One-Click-Publishing-Workflow zu ermöglichen.

4. Backend Architecture

Die Applikation ist als Serverless-Architektur auf Firebase konzipiert. Das aktuelle MVP nutzt Demo-Daten im Frontend mit vorbereiteter Firebase-Integration.

4.1 Authentifizierung (Firebase Auth)

Das System unterstützt E-Mail/Passwort sowie Google-OAuth. Die aktuelle Implementierung nutzt einen simulierten Auth-Flow mit drei Demo-Benutzern:

// AuthContext.jsx - Demo-Benutzer
const DEMO_USERS = [
  { id: 'u1', email: 'admin@hubpro.de',   name: 'Max Müller',    role: 'Admin',   avatar: 'MM' },
  { id: 'u2', email: 'agentur@hubpro.de', name: 'Sarah Schmidt',  role: 'Agentur', avatar: 'SS' },
  { id: 'u3', email: 'kunde@hubpro.de',   name: 'Thomas Weber',   role: 'Kunde',   avatar: 'TW' },
]

Die login()-Funktion simuliert einen API-Call mit 800ms Verzögerung. Die loginAsRole()-Funktion ermöglicht sofortigen Demo-Zugang.

4.2 Datenhaltung (Firestore)

Firestore wird als NoSQL-Dokumentendatenbank genutzt. Die Datenstruktur ist hierarchisch aufgebaut:

/artifacts/{appId}/public/data/
    /social_posts/{postId}
    /seo_articles/{articleId}
    /keywords/{keywordId}
    /analytics/{reportId}

4.3 Cloud Functions (Optionaler Ausbau)

Für rechenintensive Aufgaben oder Webhooks von Drittanbietern werden Firebase Cloud Functions (Node.js) vorgesehen:

  • Stripe Webhooks: Abonnement-Verwaltung und Zahlungsverarbeitung
  • Image Processing: Automatische Bildoptimierung für verschiedene Plattformgrößen
  • Scheduled Publishing: Cron-basiertes Auslösen von geplanten Posts
  • Analytics Aggregation: Periodische Zusammenfassung von Performance-Daten

8. Feature Documentation

8.1 SEO-Scoring Algorithmus

Der SEO-Score wird client-seitig in Echtzeit berechnet (via useMemo). Die Funktion calculateSEOScore() prüft 6 Kriterien:

#KriteriumMax. PunkteLogik
1Keyword im Titel (H1)20title.toLowerCase().includes(keyword)
2Keyword in den ersten 10% des Textes15HTML-Tags strippen, Substring 0-10%, Keyword prüfen
3Optimale Textlänge201500-2500 Wörter = 20 Pkt, 800-1500/2500-3500 = 10 Pkt, sonst 5
4Meta-Titel optimiert15Existiert, ≤60 Zeichen, enthält Keyword
5Meta-Description optimiert15120-160 Zeichen, enthält Keyword
6Interne/Externe Links vorhanden15Prüft auf http oder <a im Content

Score-Bewertung:

  • 80-100: Exzellent — Artikel ist hervorragend optimiert
  • 60-79: Gut — Guter Ansatz mit Verbesserungspotenzial
  • 40-59: Ausbaufähig — Mehrere Bereiche optimierbar
  • 0-39: Kritisch — Grundlegende Optimierung erforderlich

8.2 Social Media Hub — 6-Phasen Workflow

Das Herzstück der Social-Media-Verwaltung ist ein Kanban-Board mit sechs Phasen:

0. Content Dump → 1. In Bearbeitung → 2. Internes Review → 3. Approval → 4. Freigegeben → 5. Published
PhaseFarbeBeschreibungAktionen
0 - Content Dump▮ GrauRohe Ideen und KonzepteErstellen, Notizen
1 - In Bearbeitung▮ BlauAktive Content-ErstellungText schreiben, Assets zuweisen
2 - Internes Review▮ GelbInterne QualitätskontrolleFeedback, Änderungen
3 - Approval▮ LilaKunden-Review PhaseKunde: Freigeben/Korrektur
4 - Freigegeben▮ GrünVom Kunden genehmigtScheduling, Publish vorbereiten
5 - Published▮ SmaragdLive auf der PlattformMetriken tracken

Drag & Drop: Posts können via HTML5 draggable/onDragStart/onDrop Events zwischen Phasen verschoben werden. Kunden können nicht draggen.

10. User Manual / Usage Instructions

10.1 Anmeldung

  1. Öffnen Sie die Anwendung im Browser.
  2. Wählen Sie einen Demo-Zugang (Admin / Agentur / Kunde) oder melden Sie sich mit E-Mail und Passwort an.
  3. Nach erfolgreicher Anmeldung werden Sie zum Performance Dashboard weitergeleitet.
Demo-Zugangsdaten:
Admin: admin@hubpro.de | Agentur: agentur@hubpro.de | Kunde: kunde@hubpro.de
Passwort: beliebig (wird im Demo-Modus nicht geprüft)

10.2 Einen neuen Social-Post erstellen

  1. Navigieren Sie zu Social Media Hub in der Sidebar.
  2. Klicken Sie oben rechts auf + Neuer Post.
  3. Geben Sie Titel, Plattform und Caption ein.
  4. Wählen Sie das geplante Datum und die aktuelle Phase.
  5. Klicken Sie auf Post erstellen.
  6. Der Post erscheint in der gewählten Kanban-Spalte.
  7. Verschieben Sie den Post per Drag & Drop zwischen Phasen.

10.3 Einen Blog-Artikel erstellen

  1. Navigieren Sie zu SEO & Web Content.
  2. Klicken Sie auf + Artikel erstellen.
  3. Geben Sie das Fokus-Keyword ein und bestätigen Sie mit Enter.
  4. Schreiben Sie den Text im Editor-Tab. Beobachten Sie, wie die Wortanzahl und Lesezeit automatisch berechnet werden.
  5. Wechseln Sie zum SEO-Analyse Tab, um den Score und die Detailanalyse zu sehen.
  6. Im Meta-Daten Tab füllen Sie Meta-Titel (max. 60 Zeichen) und Meta-Description (120-160 Zeichen) aus.
  7. Prüfen Sie die Google-Vorschau.
  8. Wählen Sie den Status und klicken Sie auf Speichern.

10.4 Feedback als Kunde geben

  1. Melden Sie sich als Kunde an.
  2. Öffnen Sie den Social Media Hub — Sie sehen nur den Approval-Bereich.
  3. Klicken Sie auf eine Post-Karte, um die Details zu öffnen.
  4. Schreiben Sie Ihren Kommentar in das Eingabefeld und senden Sie ihn ab.
  5. Klicken Sie auf Freigeben oder Korrektur erforderlich.

10.5 Dark Mode aktivieren

  1. Klicken Sie auf das Mond-Symbol ☽ im Header oben rechts.
  2. Die gesamte Oberfläche wechselt in den Dark Mode.
  3. Erneutes Klicken wechselt zurück in den Light Mode.

11. Security Architecture

11.1 Rollenbasierte Sicherheit (RBAC)

Die Sicherheit wird auf zwei Ebenen erzwungen:

EbeneMechanismusImplementierung
UI-EbeneBedingtes Renderingif (isClient) return null — verbirgt Menüpunkte und Buttons
Daten-EbeneFirebase Security RulesPrüfung ob User-Dokument Schreibberechtigung für Collection hat
Menü-EbeneagencyOnly FlagSidebar-Items mit agencyOnly: true werden für Kunden ausgeblendet
Formular-Ebenedisabled AttributInput-Felder sind für Kunden schreibgeschützt

11.2 API-Sicherheit

  • Alle externen API-Keys werden als Environment Variables auf Vercel gespeichert.
  • Keys sind niemals im Client-Code sichtbar.
  • Firebase-Konfiguration nutzt eingeschränkte Client-Side-Schlüssel.
  • Sensible Operationen laufen über Firebase Cloud Functions (Server-Side).
